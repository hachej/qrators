# We’ve raised $17M to build what comes after Git

GitButler announced a $17M Series A led by a16z, with continued participation from Fly Ventures and A Capital. Scott Chacon frames the round as funding to accelerate a rethinking of version control for modern software development, especially in a world where humans and agents collaborate on code.

## Core argument

Chacon argues that Git solved collaboration for an earlier era, but today’s workflows have outgrown the assumptions behind it:

- one person
- one branch
- one terminal
- one linear flow

He says the real bottleneck in software development is no longer writing code itself, but preserving context across tools, people, and AI agents while organizing, reviewing, and integrating changes without creating chaos.

## What GitButler is building

GitButler positions itself not as a "better git" but as new infrastructure for how software gets built next. The company’s view is that version control should do more to:

- organize changes into logical units with context
- support stacked branches and multitasking
- make undoing and reshaping work easier
- surface merge conflicts earlier
- help teams and agents stay aware of what everyone else is doing

The post points to the recently released GitButler CLI technical preview as an early expression of that direction, designed for short-lived branch and trunk-based workflows and meant to work inside existing Git projects.

## Broader vision

A recurring theme in the post is "social coding": the idea that collaboration tooling still leaves too much context outside version control, scattered across issue trackers, chat tools, pull request descriptions, and agent interactions. GitButler’s ambition is to pull more of that context into the core workflow so working together becomes easier than working alone.
