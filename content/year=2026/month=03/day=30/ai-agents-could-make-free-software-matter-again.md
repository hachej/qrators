# AI Agents Could Make Free Software Matter Again

## Summary

George London argues that AI coding agents are poised to revitalize free software by democratizing access to software customization. Rather than requiring technical expertise to exercise software freedoms, agents can modify code on behalf of non-technical users.

## The Core Argument

London contends that Richard Stallman's "four freedoms" (run, study, modify, redistribute) became largely irrelevant during the SaaS era, since users never accessed source code running on vendor servers. AI agents fundamentally change this dynamic.

**As London explains:** "When you tell Claude or Codex 'make my task manager automatically categorize tweets I save,' you're exercising Freedom 1...through a proxy."

## Historical Context

The article traces how "free software" lost cultural prominence after a 1998 rebranding to "open source." This shift removed ethical claims about user rights while retaining code-sharing practices—particularly convenient for corporations.

The SaaS explosion further marginalized the conversation. Without source code access, the GPL's copyleft protections didn't apply, as distribution wasn't required for network services.

## Practical Example

London details attempting to automate a workflow with Sunsama (a task management tool). His agent-assisted solution required reverse-engineering unofficial APIs, storing passwords as environment variables, and manually building iOS shortcuts—highlighting how closed software forces users toward fragile workarounds.

## Forward-Looking Implications

London predicts software evaluation will increasingly prioritize agent-compatibility. Companies lacking genuine competitive advantages beyond switching costs face disruption as agents reduce those barriers.

**He notes:** "Agents are about to interpret unfree software as damage...and route around it."

## Counterarguments Acknowledged

The article addresses sustainability concerns: AI agents consuming open-source code without supporting maintainers could collapse the ecosystem. Self-hosting remains operationally burdensome despite theoretical appeal.

London suggests future models will blend open-source customization with SaaS convenience rather than forcing a binary choice.
